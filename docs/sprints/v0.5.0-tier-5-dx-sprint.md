## 2. What's new in v0.5.0

### Inline citations + structured Source panel (SDE-77)

The chat system prompt now tells the LLM to mark factual statements with `[N]` (1-based, matching the position of the chunk in the retrieved-context block) and `[N][M]` for combined sources. `ChatResponse.sources` is a list of structured `Source` objects - `n`, `source_path`, `chunk_index`, `score`, `snippet` (200-char preview) - instead of bare path strings.

The SSE terminal frame carries `Source.model_dump(mode="json")` so streaming clients see the same structure once the model finishes. HTML test client renders the panel as a collapsible `<details>` block with path, score, and an italic snippet under each entry.

### Saved templates + `sdet-brain run` (SDE-79)

YAML query templates make daily one-liners cheap. The runner dispatches in-process (no HTTP) so the same MCP-style functions back the CLI as the live MCP path.

```bash
sdet-brain template list
sdet-brain template show wcag-fact-check
sdet-brain run wcag-fact-check --var aspect="port-collision"
sdet-brain run series-status --var number="01" --format json
```

Four pre-shipped templates under `examples/templates/`:

| Template | Tool | Default args |
| --- | --- | --- |
| `voice-check` | `search_voice_samples` | `topic=self-deprecating` |
| `series-status` | `multi_query_search` | `number=01` |
| `decision-history` | `search_decisions` | `topic=deploy strategy` |
| `wcag-fact-check` | `search` (sprint-reports filter) | `aspect=Round 4 multi-page` |

User-shipped templates (in `~/.sdet-brain/templates/`) win on name collision.

### `sdet-brain-chat` REPL (SDE-78)

Terminal-first chat client backed by the existing `/chat` SSE endpoint. prompt_toolkit drives input with persistent history at `~/.sdet-brain/chat_history`. httpx streams `data:` frames, tokens render incrementally. Slash commands cover the daily flow:

| Command | What it does |
| --- | --- |
| `/help` | print available commands |
| `/clear` | reset conversation history |
| `/sources` | render structured sources from the last response |
| `/save NAME` | write conversation JSON to `~/.sdet-brain/conversations/NAME.json` |
| `/load NAME` | restore from JSON |
| `/quit` / `/exit` | leave the REPL |

`parse_command()` is a pure function so tests cover every slash branch without prompt_toolkit or network I/O.

## 4. Lessons learned

### Technical

- **Pure-function command dispatchers shrink test surface area.** The REPL's `parse_command()` returns a frozen `CommandResult` instead of doing I/O directly - tests exercise every branch in 12 lines each. The actual loop becomes a thin shell that wires stdout/network around the pure core.
- **Structured Source instead of stringly-typed paths.** Adding `n`, `score`, `snippet` to the response was nearly free at the pipeline level (the data was already there from the hybrid hit) but materially better for clients: they get enough to render footnotes without making a second query.
- **In-process tool dispatch beats HTTP for CLI templates.** The `sdet-brain run` path reuses `build_default_state()` to wire the same `AppState` the FastAPI server uses, then calls the tool functions directly. Zero HTTP round-trip means the CLI works without the daemon running, and the lazy MLX caches are still shared if both run in the same process.

### Process

- **Defer-with-rationale beats half-done shipping.** SDE-80 (persistence) is the highest-value Tier 5 feature on paper, but the right way to land it is dedicated 2-3h with the schema choices considered, not jammed into the remaining minutes after the simpler items shipped. Reopen criteria documented in Linear.
- **Combine commits when artifacts share scaffolding.** SDE-78 (REPL) and SDE-79 (templates) share `tests/cli/`, the dep on prompt-toolkit (added in SDE-77's commit), and both `[project.scripts]` entries. One commit instead of two avoids two near-duplicate test/lint cycles.
