## Lessons learned

- **Docker Desktop pulls can wedge silently.** Lost ~30 minutes on a
  stuck `docker pull qdrant/qdrant`. Hard-restarting Docker Desktop
  via `osascript -e 'quit app "Docker"' && open -a Docker` cleared
  it. Worth a runbook note for the future: if a pull stalls past 30
  s with zero progress lines, restart Docker before debugging
  anything else.
- **Library docstrings lie.** `mlx-embeddings.generate()` is
  documented to return `mx.array` but actually returns a
  transformers-style `BaseModelOutput` whose `text_embeds`
  attribute carries the dense vectors. Hard-coding `text_embeds`
  keeps the vector space stable; falling back to `last_hidden_state`
  mean-pooling would silently produce a different space.
- **FastAPI + FastMCP need a state-getter callable.** The MCP
  instance is mounted on the FastAPI app *before* the lifespan
  runs, so it cannot capture `AppState` at build time. Passing
  `state_getter=lambda: app.state.app_state` resolves it lazily on
  every tool call - clean, testable, no shared globals.
- **Markdown payloads should carry their text.** A T1-05 omission:
  the chunk payload originally lacked the chunk text, leaving
  `/search` results metadata-only. Storing `payload.text` doubles
  per-point storage at this scale (cheap) and makes one-shot reads
  possible.
- **Persistent test data is dangerous.** The first cut of
  `test_init_collections_creates_primary_and_indexes` wiped the
  production `sdet_brand_v1` collection in its teardown. Refactored
  `init_collections(name=...)` so tests use a per-process disposable
  name and never touch real data. Easy to miss; worth a Tier 2 lint
  pass.
- **`tenacity.retry` does not type-check cleanly under `mypy
  --strict`.** Switched to `Retrying()` + `for attempt in
  retrying: with attempt:`; explicit, typeable, identical
  behaviour.
- **Hard-coded brand corpus paths in two CLIs.** Both
  `sdet-brain-cli` and `sdet-brain-watcher` know about the
  `/Users/dariusz/dev/darco81/...` paths. Logged a follow-up to
  lift them to `config.py` env vars before the VPS deploy.

## Decisions log (carried forward from the architecture doc)

| # | Decision | Outcome |
| --- | --- | --- |
| 1 | Vector store: Qdrant vs pgvector | Qdrant (clean data boundary, filter-first design). |
| 2 | Server: custom FastMCP vs official `mcp-server-qdrant` | Custom (need 5+ domain tools, not just `qdrant-store`/`qdrant-find`). |
| 3 | Embeddings: MLX primary + Gemini fallback | MLX `Qwen/Qwen3-Embedding-0.6B` (1024-dim) on M4 Pro - zero API cost, full privacy. |
| 4 | Local LLM (Tier 2): default Qwen3-Next-80B vs Gemma-4 26B | Default Qwen3-Next-80B (deferred). |
| 5 | Tenancy | Single tenant (Dariusz only). |
| 6 | Cross-repo with jarvis-brain | Separate repos, pattern reuse only. |
| - | Chunk size | Soft target 800 chars, 15% overlap, atomic code/tables. Real-corpus chunks land 400-830 chars. |
| - | Distance | Cosine (matches normalised dense embeddings). |
| - | Idempotency | Deterministic UUID5 IDs from `source_path#chunk_index`; SHA-256 `content_hash` short-circuits unchanged files. |

## Tier 1 follow-ups (suggested separate Linear issues)

These surfaced during Tier 1 but were intentionally out of scope:

- **Healthcheck portability:** swap the Qdrant compose probe from
  bash `/dev/tcp/...` to `wget -qO- http://localhost:6333/readyz`
  so it works under `dash`. Cosmetic only.
- **Configurable corpus paths:** lift the hard-coded source paths in
  `cli/ingest_cli.py:_build_source_config` and
  `cli/watcher_cli.py:_default_source_config` to `config.py` env
  vars (`PROJECT_KNOWLEDGE_DIRS`, `DRAFTS_DIRS`, `ARTICLES_DIRS`,
  `SPRINT_REPORTS_DIRS`).
- **Batched cache check:** when re-ingesting a directory, look up
  `content_hash` for every file in a single `points/scroll` rather
  than O(files * RTT) round-trips.
- **Small-tail merge for the chunker:** combine sub-250-char trailing
  chunks into the previous chunk if heading boundaries allow. Open
  only if retrieval quality on the real corpus shows a gap.
